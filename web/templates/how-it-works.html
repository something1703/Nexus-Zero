Sentinel Agent

Role: Anomaly Detection & Incident Orchestrator
The Sentinel is the entry point of the pipeline. It continuously monitors GCP Cloud Logging, detects error spikes and latency anomalies, creates incidents, and dispatches them through the full investigation-remediation pipeline.

MCP Tools
detect_anomalies
Queries Cloud Logging for error spikes within a configurable time window
create_incident
Creates a new incident record with severity, error count, and error signature
acknowledge_incident
Transitions incident status and dispatches to sub-agents
How It Detects
1
Scan: Queries GCP Cloud Logging for errors in the last N minutes across all registered services
2
Deduplicate: Groups errors by service_name::error_signature to prevent duplicate incidents
3
Classify: Assigns severity (critical/high/medium/low) based on error count thresholds
4
Dispatch: Creates incident and calls Detective + Historian via Archestra A2A for parallel investigation
Example Agent Thinking
Sentinel Output
"Scanning GCP Cloud Logging for anomalies‚Ä¶
 Found 12 errors across 1 service.
 Creating incident for investigation."

Detective Agent

Role: Root Cause Analysis
The Detective digs into service logs, error patterns, and deployment history to identify the root cause of an incident. It works in parallel with the Historian for faster resolution.

MCP Tools
investigate_root_cause
Correlates error patterns, stack traces, and deployment timelines
analyze_logs
Deep dives into structured logs from Cloud Logging with time window filters
check_deployments
Reviews recent deployment history for correlation with error onset
Investigation Flow
1
Collect: Pulls structured logs from the affected service for the error window
2
Correlate: Maps error frequency against deployment events and config changes
3
Identify: Pinpoints most likely root cause (bad deploy, resource exhaustion, dependency failure)
4
Report: Sends findings to Mediator for risk assessment alongside Historian's recommendations
Example Agent Thinking
Detective Output
"Analyzing service behavior patterns, error correlations,
 and deployment history to identify root cause.
 Pulling last 30 minutes of structured logs from Cloud Logging.
 Analyzing error frequency, stack traces, and correlation patterns."

Historian Agent

Role: Knowledge Base & Solution Recommender
The Historian searches the playbook database and past incident history for proven remediation strategies. It ranks solutions by confidence and learns from every resolved incident.

MCP Tools
get_recommended_solutions
Queries playbook DB for solutions matching the error signature and service type
search_past_incidents
Finds similar past incidents and their successful resolutions
update_playbook
Records successful resolutions back into the playbook for future use
How Recommendations Work
1
Match: Searches playbooks by error signature, service name, and failure mode
2
Rank: Scores solutions by historical success rate and confidence level
3
Recommend: Returns top-N solutions with action type, confidence, and source playbook
4
Learn: After resolution, records the outcome to improve future recommendations
Example Agent Thinking
Historian Output
"Searched playbook database and past incident history.
 Found 3 solutions.
 Top recommendation: rollback (confidence: 0.85)
 from 'db-connection-pool-playbook'."

Mediator Agent

Role: Risk Assessment & Safety Guardrails
The Mediator is the safety net. It evaluates blast radius, checks change freeze windows, scores rollback safety, and produces a risk verdict. It creates human-approval requests for all critical actions.

MCP Tools
produce_recommendation
Produces final verdict with safety score, blast radius, and proposed action
analyze_blast_radius
Maps service dependency graph to assess downstream impact
check_guardrails
Validates against change freeze windows, deployment velocity, and safety thresholds
Safety Assessment Flow
1
Collect Evidence: Gathers Detective's root cause and Historian's recommendations
2
Analyze Blast Radius: Checks service dependency graph for downstream services that could be affected
3
Score Safety: Computes a safety score (0-1) considering blast radius, rollback safety, and change freeze status
4
Request Approval: Creates a pending approval record for human review with full risk context
Example Agent Thinking
Mediator Output
"Risk assessment complete.
 Verdict: APPROVE_WITH_CAUTION
 Safety score: 0.75
 Blast radius: 2 downstream services
 Proposed action: rollback
 Awaiting human approval."

Executor Agent

Role: Remediation Execution & Verification
The Executor only acts after human approval. It performs rollbacks, scaling operations, restarts, and config changes on Cloud Run services. After execution, it verifies health to confirm the fix worked.

MCP Tools
execute_remediation
Executes the approved action (rollback, scale, restart) on the target service
verify_health
Checks service health post-execution to confirm successful remediation
poll_approval
Monitors pending approvals and executes once a human approves
Execution Flow
1
Poll: Watches for approved remediation actions in the audit log
2
Execute: Calls the Cloud Run API to perform the remediation (rollback, scale, restart)
3
Verify: Runs health checks on the service to confirm errors have stopped
4
Resolve: Marks the incident as resolved and records resolution time in the audit trail
Example Agent Thinking
Executor Output
"Executing approved remediation:
 Traffic for nexus-order-api shifted to
 revision 'previous'. New instances are healthy.
 Incident marked as resolved."
